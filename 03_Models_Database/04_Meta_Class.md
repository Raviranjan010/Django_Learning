# Meta Class in Models

The `Meta` inner class in a Django model is used to define logical model-level options such as ordering, table names, and human-readable names.

**Syntax:**
```python
class Student(models.Model):
    name = models.CharField(max_length=50)

    class Meta:
        ordering = ['name']
        verbose_name = "Learner"
```

## Common Meta Options

### 1. `ordering`
Defines the default ordering of the object list.
```python
ordering = ['-name'] # Descending order by name
```

### 2. `verbose_name` / `verbose_name_plural`
Sets a human-readable name for the model (singular and plural).
```python
verbose_name = "Student Record"
verbose_name_plural = "Student Records"
```

### 3. `db_table`
Overrides the default table name generated by Django (appname_modelname).
```python
db_table = 'custom_student_table'
```

### 4. `abstract`
If `True`, this model will be an **Abstract Base Class**. It won't create a database table but can be inherited by other models to share fields.
```python
abstract = True
```

### 5. `proxy`
If `True`, allows creating a **Proxy Model** that modifies behavior (methods, managers) of the original model without changing the database schema.
```python
proxy = True
```

### 6. `app_label`
If a model is defined outside of an app in `INSTALLED_APPS`, creating this label tells Django which app it belongs to.
```python
app_label = 'myapp'
```

### 7. `permissions`
Custom permissions for the model.
```python
permissions = [('can_change_status', 'Can change status')]
```

### 8. `indexes`
Defines database indexes for the model.
```python
indexes = [
    models.Index(fields=['last_name', 'first_name']),
    models.Index(fields=['first_name'], name='first_name_idx'),
]
```
